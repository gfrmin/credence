## Sub-components

### Plugin (TypeScript)

Location: `apps/openclaw-plugin/`

OpenClaw plugin registering four hooks. The plugin is a thin client вҖ” all decision logic lives in the sidecar.

- `before_tool_call`: sends `POST /evaluate` with tool name, params, recent history. Renders the sidecar's decision as an OpenClaw hook return value (block, requireApproval, or undefined).
- `after_tool_call`: sends `POST /observe` with tool name, params, error, duration, user approval. Fire-and-forget.
- `before_compaction`: sends `POST /compaction-preview` with the messages about to be compacted.
- `agent_end`: clears local history buffer.

Fail-open: if the sidecar is unreachable, the plugin warns once and proceeds without governance.

Configuration in `~/.openclaw/config.yaml`:
```yaml
plugins:
  entries:
    credence-governance:
      enabled: true
      config:
        sidecarUrl: "http://localhost:3100"
        timeoutMs: 200
```

### Sidecar (Julia)

Location: `apps/credence-governance-sidecar/`

HTTP server (HTTP.jl) on `localhost:3100`. Four endpoints:

| Endpoint | Method | Purpose |
|---|---|---|
| `/evaluate` | POST | Compute EU, run detectors, return decision |
| `/observe` | POST | Update posterior, record outcome |
| `/compaction-preview` | POST | Scan messages for instruction patterns |
| `/health` | GET | Uptime, observation count, posterior summary |

Brain state (`brain.jl`):
- Per-(tool, category) Beta posteriors, starting at Beta(1,1).
- Category inference from tool name + argument patterns: code, delete, deploy, version-control, dependency, generic.
- EU calculation: EU(proceed) = mean - (1 - mean) with CV-based escalation threshold.
- Instruction registration with self-tuning decay (10Г— retirement ratio).
- Budget table (`config/budgets.json`) for outcome classification by tool type.

Detectors (`detectors.jl`):
- `check_stationarity`: per-(tool, argument-hash) outcome variance over window K = max(2, ceil(вҲҡ(Оұ+ОІ))), threshold p*(1-p)*0.1.
- `check_no_confidence`: CV of EU(proceed) over sliding window (size 10), threshold 1/вҲҡ(Оұ+ОІ), span 5 consecutive.
- `record_outcome!`: maintains per-(tool, argument-hash) outcome history for stationarity.

Instruction patterns (`instruction_patterns.jl`): 7 regex patterns across 4 action classes (delete, deploy, privileged-exec, dependency) plus the "any-destructive" catch-all.

### Persistence (Posture-3-schema-versioned)

Location: `~/.credence/state/posterior.json`

Schema version 1. Contents:
- `user_id`: UUID4, generated on first run.
- `tool_posteriors`: per-(tool:category) alpha/beta pairs.
- `model_posteriors`: per-(model:category) alpha/beta pairs.
- `registered_instructions`: array of instruction registrations with pattern ID, action class, timestamps, approval/denial counts.

Atomic writes via rename. File permissions 0o600, directory 0o700.

## Interventions and detectors

| Intervention | Trigger mechanism | Decision value |
|---|---|---|
| Veto-and-halt | Stationarity detector (#34574) or no-confidence detector (#65550) | `halt` |
| Escalate-to-user | Instruction detector (#1084) or general EU uncertainty | `escalate` |
| Veto-and-downgrade | EU(downgrade) > EU(proceed) naturally | `downgrade` |
| Route-to-cheaper-model | Posterior supports model downgrade | `route` |

The first two are detector-triggered. The last two fire when the posterior naturally favours them вҖ” no dedicated detector, just EU maximisation.

| Detector | Issue | Fires when | Intervention |
|---|---|---|---|
| Stationarity | #34574 | Same tool+args repeated with variance below posterior-derived threshold | halt |
| Compaction-survival | #1084 | Post-compaction action matches a registered instruction class | escalate |
| No-confidence | #65550 | EU(proceed) CV exceeds threshold for 5 consecutive evaluations | halt |

## Honest scope

Nine pre-empted limitations from the Move 3 evidence document, condensed:

1. **Caching** out of scope (OAI-compatible endpoint, no user-directed prompt caching).
2. **Cost savings** limited to halt-prevented waste; no model-routing savings claimed.
3. **Failure modes** limited to three (#34574, #1084, #65550); others named but deferred.
4. **Skill injection** (CVE-2026-25253) out of architectural scope вҖ” tool-call governance, not skill-level security.
5. **Other harnesses** deferred; sidecar HTTP is harness-agnostic, plugin is OpenClaw-specific.
6. **Cross-machine state** gives approximately-the-same-agent, not identical; sync deferred.
7. **Instruction extraction** is regex (7 patterns); LLM-based extraction is v0.2.
8. **Statistical evidence** is targeted (N=1 per scenario), not publication-grade.
9. **Veto-and-downgrade** mechanism exists in EU vocabulary but no detector triggers it; demonstration doesn't exercise it.

## Deployment

Start the sidecar:
```bash
julia apps/credence-governance-sidecar/server.jl
```

Install the plugin:
```bash
cd apps/openclaw-plugin && openclaw plugins install . --link
```

Enable in OpenClaw config (`~/.openclaw/config.yaml`) with `sidecarUrl` pointing to the sidecar (default `http://localhost:3100`). The sidecar persists state to `~/.credence/state/posterior.json` and survives restarts. Set `CREDENCE_SIDECAR_PORT` to change the listen port; set `CREDENCE_STATE_DIR` to change the persistence directory.
